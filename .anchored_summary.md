## Key Decisions
- **Certification authority grade score range**: Positive integer 0–100 (decimal for fractional scores). `grade` field is a string supporting both letter grades and descriptive labels.
- **Grade overlap validation**: Done server-side in form requests using `where('grade_end', '>=', $start)->where('grade_start', '<=', $end)`.
- **Transcript session enrolment selector**: Replaces `year_of_study` filter. Session enrolment label format: "Year X Session Y Module Z - Session Name". Cumulative mode includes all session enrolments up to the selected point.
- **FK name shortening**: All new FKs use short custom names (`ca_grade_auth_fk`, `ca_grade_cb_fk`, `ca_grade_ub_fk`) to avoid MySQL 64-char limit.
- **Module filter** kept as optional refinement on marksheet tab but disabled on transcript tab (since curriculum-based unit filtering is more relevant per-session).

## Critical Context
- **Invoice model columns**: Only `amount_due` and `computed_amount` — no `paid_amount`, `balance_due`. All controller/reconciliation code already uses correct columns.
- **certification_authority_grades migration failed first attempt**: Auto-generated FK name `certification_authority_grades_certification_authority_id_foreign` (63 chars) exceeded MySQL limit. Fixed with explicit short names.
- **`db:wipe` must use `--drop-types` only once** (causes LogicException on second run). Use plain `db:wipe` for repeated drops.
- **`syncAccountBalance` formula**: `balance = totalInvoiced - totalPaid + totalRefunded - (adjustmentCredits - adjustmentDebits)` — still doesn't handle carry_forward entries.
- **Balance carry-forward not yet implemented**: Outstanding from user — needs `carryForwardBalance()` in BillingService plus updates to `syncAccountBalance` and fee statement.
- **Student results uses `myMarksheet` endpoint** not `marksheet` — student-facing endpoints are under `/my/` prefix.
- **`AcademicSessionEnrolment` has `module` column** (nullable integer) — used in transcript label, can be used to filter units by module.
