A Course can only have one active Course Curriculum at any given time.

Older curricula remain in the system for historical records, but only one is available for new admissions.

Here's a refined implementation prompt.

Admission Form Redesign

The current admission form requires users to select:

Exam Body
Level
Course
Curriculum

using dependent dropdowns. This creates unnecessary friction and exposes the underlying database relationships to users.

Business Rule

Admissions are made against a Course Curriculum.

A Course may have many Course Curricula over time, but only one Course Curriculum can be active at any given time.

Historical curricula remain in the database for existing students and reporting, but they are not selectable for new admissions.

This means that once a course is selected, there is exactly one active Course Curriculum.

New User Experience

Replace the four dependent dropdowns with a single searchable Course field.

Backend

Create a search endpoint:

GET /api/course-curricula/search?q=

The endpoint should:

Search active courses by:
name
code
initials
Only return courses that have an active Course Curriculum.
Eager load:
course
certification authority
certification level
active curriculum

Example query:

CourseCurriculum::query()
    ->where('is_active', true)
    ->whereHas('course', function ($query) use ($search) {
        $query->where('is_active', true)
              ->where(function ($q) use ($search) {
                  $q->where('name', 'like', "%{$search}%")
                    ->orWhere('code', 'like', "%{$search}%")
                    ->orWhere('initials', 'like', "%{$search}%");
              });
    })
    ->with([
        'course.certificationAuthority',
        'course.certificationLevel',
        'curriculum',
    ]);

UX Goals
One searchable field instead of four dependent dropdowns.
Fast autocomplete with debounced search.
Search by course name, code, or initials.
No disabled dropdowns or "Select level first" messages.
Automatically derive Authority, Level, and Curriculum from the selected Course Curriculum.
Reduce the admission workflow to a single search and selection while keeping the existing database structure unchanged.
